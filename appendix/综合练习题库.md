<summary>1. [单选] 数字图书馆理论模型中，「数学模型」的核心作用是？</summary>

A. 从直观视觉角度描述构成与外观  
B. 从逻辑关系角度描述架构关系  
C. 用数学方法定量描述量化关系  
D. 描述用户与资源的交互流程

**答案：C**

**解析：** 形象模型「看得到」（A）、抽象模型「想得通」（B）、数学模型「算得清」（C）。数学模型用资源规模、服务效率、技术性能等量化指标定量描述。核心维度始终是资源、服务、技术（第1章，关联第2/3/4/5/6章各项技术指标）。

A. PUT  
B. PATCH  
C. POST  
D. DELETE

**答案：B**

**解析：** PATCH用于部分更新资源（如只修改用户邮箱）。PUT是整体更新，POST是新建/提交，DELETE是删除。GET是只读。HTTP连接建立经三次握手、断开经四次挥手（第2章）。

**解析：** 「不可更新」指数据存入后不再修改，而是定期追加新数据，保留历史轨迹。这是数据仓库区别于操作型数据库的重要特征（第2章，关联第3章存储与备份）。

C. 资源不存在  
D. 服务器内部错误

**答案：C**

A. INSERT INTO Student VALUES (...)  
B. DELETE FROM Student WHERE ...  
C. UPDATE Student SET age=23 WHERE name='李四'  
D. SELECT * FROM Student

**答案：C**

<summary>2. 简述数字图书馆的产生背景及其三类驱动因素。</summary>

**参考答案：**

数字图书馆的产生背景可以从三个层面理解：

1. **内部驱动力：** 图书馆自身发展产生的内在需求。
   - 馆藏数字化的需求（纸质文献→数字资源）
   - 服务模式升级的需要（到馆服务→网络服务）
   - 从图书馆自动化向数字化的转型

2. **外部推动力：** 外部环境对数字图书馆建设的推动。
   - 计算机与网络技术的进步（互联网普及、存储技术发展）
   - 国家政策的支持（信息化建设战略）
   - 信息产业的市场竞争

3. **社会环境：** 社会信息化发展的大环境。
   - 信息爆炸时代（数据量急剧增长）
   - 知识经济的兴起（知识成为核心生产要素）
   - 数字鸿沟问题（信息获取不平等）

**总结：** 内部驱动力是「我想做」，外部推动力是「外部推我做」，社会环境是「时代需要我做」。三者共同推动了数字图书馆的诞生和发展。

**参考答案：**

动机机理回答「数字图书馆为什么会存在和发展」，包括两类动力：

1. **运行动力：** 维持数字图书馆日常运行和持续发展的动力。具体表现为：技术更新驱动（新技术不断引入）、用户需求拉动（读者对便捷服务的需求）、管理机制保障（组织与制度的支撑）。

2. **学术动力：** 推动数字图书馆学术研究和理论创新的动力。具体表现为：学科发展需要（图书情报学自身演进）、学术交流促进（国内外学术共同体互动）、研究成果推动（科研产出的转化应用）。

**理解要点：** 运行动力是「活下去」的动力，学术动力是「走得更远」的动力。二者相辅相成，共同推动数字图书馆从建设走向持续发展。

</details>

<details>
<summary>4. 简述数字图书馆系统的7项安全需求。</summary>

**参考答案：**

数字图书馆系统的7项安全需求：

1. **信息保密性** —— 信息不被未授权者获取
2. **信息的不可否认性** —— 发送方不能否认已发送的信息
3. **管理员和读者身份的认证性** —— 确认用户身份的合法性
4. **系统的可靠性** —— 系统持续稳定运行
5. **可访问性** —— 合法用户能够正常访问系统
6. **防御性** —— 系统能够抵御攻击
7. **合法性** —— 信息操作符合法律法规

</details>

<details>

**参考答案：**

围绕7项安全需求，数字图书馆采用的安全技术主要包括四类：

1. **防火墙：** 在内部网与外部网之间设置隔离屏障，按安全策略过滤进出流量，是最基础的网络边界防护手段。

2. **加密技术：** 分为对称加密（加解密同一密钥，速度快）、非对称加密（公钥加密/私钥解密，安全性高但速度慢）、混合加密（用非对称加密协商会话密钥、再用对称加密传输数据，兼顾安全与效率）。

3. **数字签名与消息摘要：** 用发送方私钥对消息摘要加密形成数字签名，实现身份认证和不可否认性；消息摘要（如MD5/SHA）保证数据完整性。

4. **数字证书与PKI：** 由可信的认证中心（CA）签发数字证书，将公钥与持有者身份绑定；PKI（公钥基础设施）提供证书签发、管理、验证、撤销的完整体系，是电子商务和数字图书馆信任的基础。

</details>

<details>
<summary>6. 简述系统集成在数字图书馆中的三种类型。</summary>

**参考答案：**

数字图书馆中的系统集成包括三种类型：

1. **数据集成：** 不同数据源间的数据移植和整合。实现方式包括数据复制、数据聚合、接口集成。

2. **业务过程集成：** 业务管理、进程模拟、工作流。将图书馆的各项业务流程进行整合，实现跨部门、跨系统的业务协同。

3. **服务集成：** 通过框架、事务、分布式对象等提供内外应用集成。基于Web Server机制，实现不同应用系统之间的服务调用和互操作。

</details>

<details>
<summary>7. 简述HTTP协议中常见的请求方法及其作用。</summary>

**参考答案：**

HTTP协议常见的请求方法有7种，对应不同的操作语义：

| 方法 | 作用 | 典型场景 |
|------|------|---------|
| GET | 获取资源（只读） | 浏览器打开网页 |
| POST | 新建资源或提交数据 | 提交注册表单 |
| PUT | 新建或整体更新资源 | 修改用户完整信息 |
| PATCH | 部分更新资源 | 只修改用户邮箱 |
| DELETE | 删除资源 | 删除留言 |
| HEAD | 只获取响应头 | 检查文件大小 |
| OPTIONS | 查询服务器支持的方法 | 跨域CORS预检 |

**记忆要点：** GET只读、POST新建/提交、PUT整体更新、PATCH部分更新、DELETE删除。HTTP连接建立经过「三次握手」，断开经过「四次挥手」。

</details>

<details>
<summary>8. 简述数据仓库的定义及其四个基本特征。</summary>

**参考答案：**

**定义：** 数据仓库（Data Warehouse, DW）是一个面向主题的、集成的、不可更新的、随时间不断变化的数据集，用以更好地支持企业或组织的决策分析处理。

**四个基本特征：**

1. **面向主题：** 数据按业务主题（如读者、馆藏、借阅）组织，而非按具体应用流程组织。
2. **集成的：** 将多个异构数据源的数据经过抽取、清洗、转换后统一加载，消除不一致。
3. **不可更新：** 数据一旦存入数据仓库后不再修改，而是定期追加新数据，保留历史轨迹。
4. **随时间不断变化：** 数据带有时间属性，反映不同时间点的状态，支持趋势分析。

**与操作型数据库的区别：** 操作型数据库面向日常事务处理，频繁增删改；数据仓库面向决策分析，以查询为主、不可更新。

</details>

<details>
<summary>9. 简述数据挖掘的五大基本任务。</summary>

**参考答案：**

数据挖掘的五大基本任务：

1. **分类与预测：** 将数据划分到预定义的类别中，或预测连续型变量的值（如预测读者是否会逾期）。
2. **聚类分析：** 按相似性将数据分组，组内相似度高、组间差异大，无需预定义类别（如读者群体细分）。
3. **关联规则挖掘：** 发现数据项之间的关联关系，典型应用是购物篮分析（如借阅A书的人常借B书）。
4. **序列模式挖掘：** 发现随时间顺序出现的事件模式（如检索行为的时序规律）。
5. **异常检测：** 识别与大多数数据显著不同的数据项，用于发现异常借阅或攻击行为。

</details>

<details>
<summary>10. 简述语义搜索的5步工作流程。</summary>

**参考答案：**

语义搜索基于向量检索，其5步工作流程为：

1. **查询分析：** 将查询分解为关键词、实体和短语，涉及分词、词性标注和命名实体识别等NLP方法。
2. **向量嵌入：** 将关键词和短语投射到高维向量空间中，使语义相似的词在空间中距离更近。
3. **计算向量相似度：** 通过余弦相似度、欧氏距离等方法计算查询向量与文档向量的相似度。
4. **知识图谱：** 创建包含语义信息的结构化网络，将文本表示为节点和边，添加上下文和推理层。
5. **结果重新排序：** 根据语义相关性、用户上下文和偏好对结果重新排序后输出。

</details>

<details>
<summary>11. 简述数据压缩的可行条件。</summary>

**参考答案：**

数据压缩之所以可行，基于三个前提条件：

1. **信息中包含冗余信息：** 数据中存在重复或可预测的部分，可以去掉冗余而不损失信息（无损压缩的基础）。
2. **相邻数据间存在关联性：** 相邻的数据样本之间存在统计相关性，可以利用这种关联进行预测编码。
3. **人们对信号的时间和幅度变化有一定的极限：** 人类视觉、听觉对信号的感知有阈值和局限，可以舍弃感知不到的细节（有损压缩的基础）。

**理解要点：** 前两条支撑无损压缩，第三条支撑有损压缩。三者共同决定了数据压缩的理论可行边界。

</details>

<details>
<summary>12. 简述混合加密的工作流程。</summary>

**参考答案：**

在真实系统中，通常采用**混合加密**方式，结合非对称加密的安全性和对称加密的速度优势。其流程为：

1. **验证证书：** 客户端通过CA签发的数字证书验证服务器身份，获取服务器公钥。
2. **非对称加密协商会话密钥：** 客户端用服务器公钥加密一个随机的会话密钥并发送，只有服务器私钥能解密，从而安全地协商出会话密钥。
3. **对称加密传输数据：** 双方使用协商好的会话密钥对称加密传输业务数据，速度快、效率高。
4. **密钥失效：** 会话结束后会话密钥失效，下次通信重新协商新密钥。

</details>

<details>
<summary>13. 简述推荐系统的闭环工作流程。</summary>

**参考答案：**

推荐系统的工作流程是一个闭环：

1. **数据采集** —— 收集用户的借阅、检索、点击、课程等行为数据
2. **特征建模** —— 构建用户画像、资源画像、场景画像
3. **算法计算** —— 使用推荐算法（协同过滤、基于内容等）计算推荐结果，进行相似度计算、评分预测、排序
4. **结果呈现** —— 将推荐结果以列表、解释、提醒等形式呈现给用户
5. **反馈优化** —— 收集用户对推荐结果的反馈（点击、借阅、忽略、评价），用于优化下一轮推荐

**闭环特点：** 数据进入系统后产生推荐结果，推荐结果反过来产生新的用户行为数据，形成不断优化的闭环。

</details>

<details>
<summary>14. 简述协同过滤的两种类型及其特点。</summary>

**参考答案：**

协同过滤是推荐系统的核心方法之一，分为两种类型：

1. **User-based协同过滤（基于用户的协同过滤）：** 找到与目标用户兴趣相似的其他用户，把相似用户喜欢而目标用户尚未接触的资源推荐给目标用户。
   - 逻辑：「和你相似的人也喜欢」
   - 优点：能发现用户的新兴趣
   - 缺点：用户数据稀疏时效果下降

2. **Item-based协同过滤（基于资源的协同过滤）：** 找到与用户已借阅资源相似的其他资源进行推荐。
   - 逻辑：「和你借过的书相似的书」
   - 优点：资源相对稳定，计算可预先缓存，适合资源数远少于用户数的场景
   - 缺点：倾向于推荐相似资源，可能造成推荐趋同

**对比基于内容的推荐：** 基于内容推荐依赖资源特征相似（越推越窄），协同过滤依赖用户行为相似（能发现新兴趣但有冷启动问题）。

</details>

<summary>1. 论述数字图书馆从「产生背景」到「理论模型」再到「核心技术」的完整发展逻辑。</summary>

数字图书馆的理论模型以资源、服务、技术为核心维度，课程中的各项技术分别支撑这三大维度：

- 数据压缩（无损/有损/混合）—— 提高存储效率，降低传输成本

**服务维度（检索与推荐）：**
- 基于内容的检索（TF-IDF/分词）—— 基础检索服务
- 语义搜索（向量嵌入/知识图谱）—— 智能化检索服务
- 跨语言检索（CLIR）—— 跨语言知识服务

**总结：** 资源是基础，服务是目标，技术是手段。三者相互依存，共同构成数字图书馆的完整体系。

<details>
<summary>3. 论述基于内容的检索与语义搜索的区别，并说明从前者到后者的发展脉络。</summary>

**参考答案：**

**一、二者区别：**

| 维度 | 基于内容检索 | 语义搜索 |
|------|------------|---------|
| 匹配方式 | 关键词字面匹配 | 理解查询意图和上下文含义 |
| 对同义词 | 无法识别（如"电脑"与"计算机"匹配不上） | 向量空间中距离近，可识别 |
| 对歧义 | 无法消歧（"苹果"是水果还是公司） | 结合上下文消歧 |
| 技术复杂度 | 低（TF-IDF/倒排索引） | 高（ML+NLP+向量检索+知识图谱） |
| 资源消耗 | 少 | 多 |
| 个性化 | 弱 | 强 |

**二、从基于内容到语义搜索的发展脉络：**

1. **基于内容检索阶段：** 以TF-IDF为代表，通过词频统计衡量文档内重要性，用IDF衡量跨文档区分度。优点是简单高效，局限是无法理解同义、歧义和用户意图。

2. **向量嵌入阶段：** 将文本投射到高维向量空间。从One-hot（稀疏、无法算相似度）→ word2vec（密集固定向量，基于共现）→ BERT（上下文相关的动态向量，捕获深层语义）→ LLM（最强语义表示）。词向量的演进使"语义相似"可计算。

3. **语义搜索阶段：** 在向量嵌入基础上，结合查询分析、相似度计算（余弦等）、知识图谱（添加上下文和推理）、结果重排序，形成完整的5步工作流程。

4. **检索增强阶段（RAG）：** 用检索结果增强大语言模型，补充LLM不知道的知识、减少幻觉，使数字图书馆从"信息平台"升级为"智能知识服务平台"。

**三、总结：** 发展的本质是从"字面匹配"走向"语义理解"，从"找关键词"走向"理解意图"，技术驱动则是从统计方法走向深度学习和大模型。这一发展使数字图书馆能够回答"适合新手学习人工智能的书"这类模糊、探索式的查询。

</details>

<details>
<summary>4. 论述推荐系统的工作流程、主要算法、评估指标与风险治理。</summary>

**参考答案：**

**一、工作流程（闭环）：**

推荐系统是一个不断优化的闭环：数据采集 → 特征建模 → 算法计算 → 结果呈现 → 反馈优化。数据进入系统产生推荐结果，推荐结果又产生新的用户行为数据，形成正反馈闭环。

**二、主要算法：**

| 算法 | 逻辑 | 优点 | 缺点 |
|------|------|------|------|
| 基于内容推荐 | 资源特征相似 | 解释性强、无冷启动 | 越推越窄 |
| 协同过滤(User-based) | 找相似用户 | 发现新兴趣 | 用户稀疏时失效 |
| 协同过滤(Item-based) | 找相似资源 | 可预计算、稳定 | 推荐趋同 |
| 基于知识推荐 | 利用领域知识 | 可解释、适合冷启动 | 知识工程成本高 |
| 混合推荐 | 组合多种方法 | 互补、综合最优 | 系统复杂度高 |

**三、评估指标：**

主要从准确性和排序质量两方面评估：
- 准确率/召回率/F1 —— 衡量推荐结果的相关性
- MAP（平均精度均值）—— 衡量排序的整体质量
- NDCG（归一化折损累计增益）—— 衡量排序位置加权后的质量
- 覆盖率/多样性 —— 衡量推荐结果的广度
- 用户满意度 —— 主观体验指标

**四、风险与治理：**

推荐系统面临的主要风险包括：信息茧房（用户被困在兴趣圈）、过滤气泡、偏见放大、隐私泄露、冷启动问题。对应的治理原则包括：保障用户控制权、算法透明可解释、保护用户隐私、提供多样性、建立反馈与纠偏机制等。治理的目标是让推荐"既准又多元、既智能又可信"。

**五、总结：** 推荐系统使数字图书馆从"被动响应检索"走向"主动推荐知识"，是数字图书馆服务升级的关键技术；但其风险也要求配套治理，确保技术向善。

</details>
