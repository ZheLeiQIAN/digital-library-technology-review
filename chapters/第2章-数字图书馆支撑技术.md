# 第2章 数字图书馆支撑技术

本章是数字图书馆的技术基础模块，涵盖Web技术、软件架构、数据仓库、系统集成和安全技术。这些通用信息技术是构建数字图书馆系统的"地基"和"水电系统"。

## 【核心概念】

### 2.1 Web技术 ★

**Web特征：**
1. 以超文本和多媒体形式存在的网络信息空间
2. 与平台无关
3. 提供直观与易于使用的图形界面
4. 分布式的
5. 动态的、交互的

**Web体系结构四大要素：**
- 客户机/Web服务器
- 统一资源定位器（URL）
- HTTP协议
- 数据库技术

**URL结构：** (1) 协议/机制 (2) 页面所在机器的域名 (3) 页面标识

**常见URL协议：**

| 名字 | 用途 |
|------|------|
| HTTP | 超文本传输协议 |
| FTP | 文件传输协议 |
| FILE | 本地文件 |
| News | 新闻组或新闻文章 |
| Gopher | Gopher查找信息 |
| Telnet | 远程登录 |
| MAILTO | 发送电子邮件 |
| WAIS | Wide Area Information Server 广域信息服务 |

**HTTP协议常见请求方法：** ★

| 方法 | 作用 | 示例 |
|------|------|------|
| GET | 获取资源（只读） | 浏览器打开网页 |
| POST | 新建资源或提交数据 | 提交注册表单 |
| PUT | 新建或整体更新资源 | 修改用户完整信息 |
| PATCH | 部分更新资源 | 只修改用户邮箱 |
| DELETE | 删除资源 | 删除留言 |
| HEAD | 只获取响应头 | 检查文件大小 |
| OPTIONS | 查询服务器支持的方法 | 跨域CORS预检 |

**HTTP连接：三次握手；断开：四次挥手**

**HTTP常见状态码：**

| 状态码 | 类别 | 含义 | 场景示例 |
|--------|------|------|----------|
| 200 | 成功 | OK，请求成功处理 | 正常打开网页 |
| 301 | 重定向 | Moved Permanently，永久重定向 | 网站更换域名 |
| 302 | 重定向 | Found，临时重定向 | 短期维护跳转 |
| 400 | 客户端错误 | Bad Request，请求语法错误 | 参数格式错误 |
| 401 | 客户端错误 | Unauthorized，未授权 | 未登录访问受限资源 |
| 403 | 客户端错误 | Forbidden，禁止访问 | 权限不足 |
| 404 | 客户端错误 | Not Found，资源不存在 | 网页不存在或URL错误 |
| 500 | 服务器错误 | Internal Server Error，内部服务器错误 | 服务器程序异常 |
| 502 | 服务器错误 | Bad Gateway，网关错误 | 代理服务器无法连接后端 |
| 503 | 服务器错误 | Service Unavailable，服务不可用 | 服务器过载或维护 |

### 2.2 数据库技术 ★

**数据库在数字图书馆中的作用：**
1. 存储与管理各种数据
2. 决策支持
3. 建立综合信息服务系统

**SQL基本操作（增删改查）：**

| 操作 | SQL语句示例 |
|------|------------|
| 增(INSERT) | `INSERT INTO Student (id, name, age, major) VALUES (4, '赵六', 22, '数据科学');` |
| 删(Delete) | `DELETE FROM Student WHERE id = 3;` |
| 改(Update) | `UPDATE Student SET age = 23 WHERE name = '李四';` |
| 查(Read) | `SELECT * FROM Student` |
| 条件查询 | `SELECT * FROM Student where major = '计算机'` |
| 投影查询 | `SELECT name, age, major FROM Student` |
| 排序 | `ORDER BY` 子句 |
| 分页 | `LIMIT <N-M> OFFSET <M>` |
| 聚合 | `SELECT COUNT(*) FROM students;` |
| 连接查询 | `JOIN` 语句 |

**Web数据库接口技术：**
1. **CGI** (Common Gateway Interface)：Web服务器与数据库之间的中介
2. **ODBC** (Open Database Connectivity)：开放式数据库连接，允许程序使用SQL作为数据访问标准
3. **JDBC** (Java Database Connectivity)：Java数据库连接，HTML中嵌入Java applet调用JDBC访问数据库

### 2.3 B/S结构与C/S结构 ★

| 维度 | B/S结构（浏览器/服务器） | C/S结构（客户端/服务器） |
|------|------------------------|------------------------|
| 定义 | 最流行的网络软件系统结构 | 经典的分布式计算模型 |
| 客户端 | Web浏览器 | 本地安装的特定应用程序 |
| 优势 | 部署成本低、易于访问、跨平台、易扩展、数据集中管理 | 用户体验丰富、本地处理能力强、安全性更高、网络通信高效 |
| 劣势 | 用户体验有限、高度依赖网络、服务器压力大 | 部署维护成本高、跨平台性差、升级困难 |
| 应用场景 | 互联网应用主流 | 需要复杂本地处理的场景 |

### 2.4 中间件 ☆

**定义：** 中间件（Middleware）是一种独立的系统软件服务程序，分布式应用软件借助这种软件在不同的技术之间共享资源，位于客户机服务器的操作系统之上，管理计算资源和网络通信。

**常用中间件类型：**

| 类型 | 功能 |
|------|------|
| 分布式中间件 | 协调多个服务器之间的任务分工和通信；任务调度、RPC、服务注册与发现 |
| 负载均衡中间件 | 把用户请求平均分配到多台服务器，防止单点压力过大 |
| 缓存中间件 | 把经常访问的数据暂存到高速存储（内存），提升访问速度 |
| 数据库中间件 | 分库分表、读写分离、SQL路由 |

### 2.5 数据仓库 ★

**定义：** 数据仓库（Data Warehouse, DW）是一个面向主题的、集成的、不可更新的、随时间不断变化的数据集，用以更好地支持企业或组织的决策分析处理。

**四个基本特征：**
1. **面向主题的** —— 围绕主题组织数据
2. **集成的** —— 从多个数据源整合
3. **不可更新的** —— 一旦存入，不再修改
4. **随时间不断变化的** —— 定期追加新数据

### 2.6 数据挖掘 ☆

**定义：** 数据挖掘（Data Mining）是通过特定算法对大量数据进行处理和分析以发现数据中的模式、趋势或关联性的过程。也被称作知识发现（Knowledge Discovery in Database, KDD）。

**五大基本任务：**

| 任务 | 定义 |
|------|------|
| 分类与预测 | 根据已知数据特征，将数据项划分到预先定义的类别中 |
| 聚类分析 | 在没有预先定义类别的情况下，按相似性分组 |
| 关联规则挖掘 | 发现数据项之间的有趣关系（如购物篮分析） |
| 序列模式挖掘 | 发现数据项之间的时间序列关系 |
| 异常检测 | 发现与大多数数据项显著不同的数据项 |

**对数字图书馆的意义：** 数据收集存储组织、数据分析与知识挖掘、决策支持（读者查询日志挖掘、图书借阅数据挖掘）。

### 2.7 集成技术 ☆

**系统集成定义：** 根据应用需求，将硬件平台、网络设备、系统软件、工具软件及应用软件等集成为具有优良性价比的计算机系统的全过程。

**数字图书馆中的三种集成：**
1. **数据集成：** 不同数据源间的数据移植和整合（数据复制、数据聚合、接口集成）
2. **业务过程集成：** 业务管理、进程模拟、工作流
3. **服务集成：** 通过框架、事务、分布式对象等提供内外应用集成（Web Server机制）

### 2.8 安全技术 ★

**数字图书馆系统的安全需求（7项）：**
1. 信息保密性
2. 信息的不可否认性
3. 管理员和读者身份的认证性
4. 系统的可靠性
5. 可访问性
6. 防御性
7. 合法性

**安全常用方法：**

**防火墙（Firewall）技术：**
- 内部网和外部网之间的安全防范系统
- 安全策略：(1) 没有被列为允许访问的服务都是禁止的；(2) 没有被列为禁止访问的服务都是允许的

**加密技术：**

| 类型 | 特点 | 流程 |
|------|------|------|
| 对称加密 | 发送方和接收方共用一把密钥；速度快 | 明文 → 加密 → 密文传输 → 解密 → 明文 |
| 非对称加密 | 一对密钥（公钥公开、私钥保密）；适合建立信任 | A用B的公钥加密 → B用自己的私钥解密 |
| 混合加密 | 真实系统常用：非对称加密协商会话密钥 + 对称加密传输数据 | 验证证书 → 非对称加密协商密钥 → 对称加密传输 → 密钥失效 |

**消息摘要（Message Digest）与数字签名（Digital Signature）：**
- 消息摘要：Hash加密算法，摘要成为消息的"指纹"
- 数字签名：消息的接受方能够确认或证实发送方的签名

**数字证书（Digital Certificate）：** 由证书机构(CA)签署的由用户公钥、身份信息及认证中心签名构成的实体。

**PKI技术** (Public Key Infrastructure，公钥基础设施)：管理用户的数字证书、公钥及安全政策的一系列安全服务的集合。包括：认证中心、证书库、证书撤销、密钥备份与恢复、自动密钥更新、密钥文档管理、交叉认证、数字时间戳、客户端软件。

## 【重点公式/算法】

无特定数学公式，重点掌握各种技术的原理和流程。

## 【方法比较】

| 维度 | B/S | C/S |
|------|-----|-----|
| 部署维护 | 成本低 | 成本高 |
| 跨平台 | 好 | 差 |
| 用户体验 | 一般 | 丰富 |
| 安全性 | 挑战大 | 更高 |
| 升级 | 容易 | 困难 |

| 加密类型 | 密钥数量 | 速度 | 适用场景 |
|---------|---------|------|---------|
| 对称加密 | 1把共享密钥 | 快 | 大量数据传输 |
| 非对称加密 | 公钥+私钥 | 慢 | 建立信任、交换密钥 |
| 混合加密 | 结合两者 | 综合 | 真实系统标准做法 |

B. PUT

**答案：C**

**解析：** PATCH用于部分更新资源（比PUT更精细），例如只修改用户的邮箱而不影响其他信息。GET是获取资源，PUT是整体更新，DELETE是删除资源。

<summary>【练习2】多选题：数据仓库的基本特征包括哪些？</summary>

A. 面向主题的  
B. 集成的  
C. 可实时更新的  
D. 随时间不断变化的  
E. 不可更新的

**答案：A、B、D、E**

**解析：** 数据仓库的四个基本特征是：面向主题的、集成的、不可更新的、随时间不断变化的。注意是"不可更新"而非"可实时更新"，数据仓库一旦存入不再修改，定期追加新数据。

<details>
<summary>【练习3】简答题：简述B/S结构与C/S结构的主要区别。</summary>

**参考答案：**

| 维度 | B/S结构 | C/S结构 |
|------|---------|---------|
| 全称 | 浏览器/服务器 | 客户端/服务器 |
| 客户端 | Web浏览器 | 本地安装的特定应用程序 |
| 部署维护 | 成本低，易于扩展 | 成本高，升级困难 |
| 跨平台 | 良好 | 差 |
| 用户体验 | 受限于浏览器 | 丰富，本地处理能力强 |
| 网络依赖 | 高度依赖网络 | 可离线处理部分功能 |
| 安全性 | 面临更多安全挑战 | 相对更高 |

B/S是当今互联网应用的主流架构，C/S适合需要复杂本地处理的场景。

</details>

<details>
<summary>【练习4】填空题：在真实系统中，通常采用____加密方式，即先用____加密安全协商会话密钥，再用____加密高速传输数据。</summary>

**答案：混合；非对称；对称**

**解析：** 混合加密结合了两种加密方式的优点：非对称加密适合"建立信任、交换密钥"但速度相对慢；对称加密适合"传输大量数据"速度快，但需要先解决密钥共享问题。真实系统的流程是：验证证书 → 非对称加密协商会话密钥 → 对称加密传输数据 → 会话结束密钥失效。

</details>

<details>
<summary>【练习5】论述题：简述数字图书馆系统安全常用的方法及其原理。</summary>

**参考答案：**

1. **防火墙技术：** 内部网和外部网之间的安全防范系统，一种访问控制机制。安全策略包括：(1)没有被列为允许访问的服务都是禁止的；(2)没有被列为禁止访问的服务都是允许的。

2. **加密技术：**
   - **对称加密：** 发送方和接收方共用一把密钥，加密解密速度快，适合大量数据传输。
   - **非对称加密：** 一对密钥（公钥公开、私钥保密），适合建立信任和交换密钥。
   - **混合加密：** 真实系统常用，先用非对称加密协商会话密钥，再用对称加密高速传输数据。

3. **消息摘要：** 使用Hash算法生成消息的"指纹"，用于验证数据完整性。

4. **数字签名：** 让消息的接受方能够确认发送方的身份。

5. **数字证书与PKI：** 由证书机构(CA)签署，包含用户公钥和身份信息；PKI是管理数字证书、公钥及安全政策的集合。

</details>
